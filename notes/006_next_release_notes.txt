Next release notes draft
========================

Testing and correctness evidence
--------------------------------

- Expanded the pinned RDKit parity suite around exact, version-keyed fixtures.
- Moved RDKit-derived expected outputs out of inline Python constants and into
  fixture families by claim type:
  - exact support / inventory equality
  - serializer regressions
  - deterministic writer-output membership
  - rooted random-writer outputs
  - disconnected sampling compatibility
  - stereo regression membership checks
- Strengthened fixture loaders so malformed IDs, duplicate cases, bad
  `rdkit_version` values, string/integer/boolean coercion, duplicate expected
  outputs, and malformed compatibility fixtures fail early.
- Added direct fixture-loader contract tests, including negative schema tests
  for compatibility fixtures.
- Updated the pinned RDKit parity runner so it fails when the installed RDKit
  version has no checked-in pinned fixture corpus, instead of silently passing
  via skipped tests.
- Added fixture taxonomy documentation in `docs/testing-fixtures.md`.

Scope note
----------

This is primarily test-suite and release-confidence infrastructure.  It does
not intentionally change public runtime behavior.
