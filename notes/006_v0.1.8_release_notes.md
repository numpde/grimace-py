# grimace-py v0.1.8

This release focuses on correctness evidence, RDKit parity coverage, and
release confidence.  It includes small reference-preparation parity fixes, but
does not intentionally broaden the supported public runtime beyond the
documented `canonical=False, doRandom=True` subset.

Highlights

- Added a pinned RDKit parity lane keyed by exact `rdBase.rdkitVersion`; pinned
  parity now fails if the installed RDKit version has no checked-in corpus.
- Expanded the exact RDKit fixture corpus across serializer edge cases,
  rooted writer behavior, deterministic writer-output membership, disconnected
  molecules, stereo membership, atom maps, isotopes, directional bonds,
  explicit hydrogens, metals, molblock-backed cases, and larger sampled
  regressions.
- Moved RDKit-derived expected outputs out of inline Python constants and into
  JSON fixture families grouped by claim type and serializer feature.
- Strengthened fixture loaders and contract tests so malformed IDs, duplicate
  cases, bad `rdkit_version` values, string/integer/boolean coercion,
  duplicate expected outputs, malformed compatibility fixtures, and missing
  pinned corpora fail early.
- Added exact public-invariant and installed-artifact correctness runners, so
  CI exercises broader public API identities and installed wheel/sdist paths.
- Fixed reference-preparation parity for `ignoreAtomMapNumbers` and isotope
  suppression in non-isomeric output.
- Documented the fixture taxonomy in `docs/testing-fixtures.md` and clarified
  that loaders, contract tests, parity runners, and CI are the enforcement
  layer; docs explain intent only.

Known scope

- The public runtime remains the documented strict subset:
  `canonical=False, doRandom=True`.
- Exact RDKit parity evidence is version-keyed.  Updating RDKit may require new
  pinned fixture expectations.
- `pip install grimace` still installs an unrelated package from PyPI.  Install
  from the GitHub release assets instead.
