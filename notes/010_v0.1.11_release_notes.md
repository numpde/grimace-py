# grimace-py v0.1.11

This release adds public prepared-molecule serialization and routes the public
runtime through that prepared representation.

Highlights

- Added `grimace.PrepareMol(...)` and opaque `grimace.PreparedMol` byte
  round trips via `to_bytes()` and `PreparedMol.from_bytes(...)`.
- `PreparedMol` is accepted by enumeration, both decoders, token inventories,
  and deviation diagnostics when writer flags match.
- Public RDKit molecule inputs now normalize through `PrepareMol(...)` before
  runtime execution, keeping RDKit work at the preparation boundary.
- Moved prepared-molecule storage and byte decoding into Rust, with a compact
  versioned binary payload.
- Moved prepared static token-inventory superset logic to Rust.
- Added `MolToSmilesDeviation(...)` for diagnosing the first unsupported
  character or external-token boundary in a candidate serialization.
- Tightened option metadata, writer-flag validation, reference policy parsing,
  and prepared graph metadata validation.
- Refreshed the timing table and adjusted the opt-in stereo perf assertion to
  cover the intended high-support regime.

Validation

- `PYTHONPATH=python:. python -m unittest discover -s tests -t . -q`
- `RUN_PERF_TESTS=1 PYTHONPATH=python:. python -m unittest tests.perf.test_kernel_nonstereo tests.perf.test_kernel_stereo tests.perf.test_readme_timings -v`
- `cargo test -q`
