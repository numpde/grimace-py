# South Star Release-Readiness Audit

Branch: `south-star`

Date: 2026-05-21

Task: `South Star 129: Run South Star release-readiness audit`

## Verdict

Do not promote `MolToSmilesEnumS` to a public package API yet.

The branch is substantially more coherent than the initial South Star seed:
the private boundary is documented, unsupported cases fail fast, diagnostics
are richer, and the graph-native path no longer depends on RDKit writer support
as generation input. That is enough for continued internal iteration, but not
enough for a package-ready semantic enumerator.

The blocking issue is not failing tests. The blocking issue is authority: no
fixture domain is currently marked `unified-reference-backed`, and the
readiness matrix still reports every checked case as blocked for public API
promotion.

## Blockers

### 1. No Unified-Reference Authority Yet

No support fixture is currently backed by the final one-truth reference model.
All checked cases are still temporary-witness or regression-witness authority.

This blocks public export even where graph-native generation already matches
the expected support set.

Follow-up: define and implement the first domain that can honestly move from
temporary witness authority to unified-reference authority.

### 2. Some Cases Still Bypass The Shared Pipeline

Twelve checked cases are not generated by the shared fact/event/constraint/
renderer pipeline according to the readiness matrix.

This does not imply the cases are wrong. It means they cannot be claimed as
package-ready support until the missing spine stages are explicit or the cases
are classified as intentionally out of the checkpoint scope.

Follow-up: enumerate those cases, classify the missing stage, and create one
implementation row per structural blocker.

### 3. The Public API Surface Is Still A Decision, Not An Implementation Step

The private contract is now inspectable, but export remains a gate. The branch
should not export `MolToSmilesEnumS` until the support authority problem is
resolved or the project deliberately chooses a narrower experimental export
with an explicit maturity label.

Follow-up: keep the export gate as a Decision row after the post-checkpoint
roadmap is synthesized.

### 4. Complexity Evidence Is Guardrail-Level, Not Release-Level

The current complexity fixture guards representative counts and catches hidden
generate/filter growth, but it is not yet a release performance claim. It is
adequate for internal readiness and inadequate for a user-facing performance
story.

Follow-up: keep package release notes away from performance claims for EnumS
until there is a separate benchmark plan for the semantic enumerator.

## Recommendation

Proceed with internal South Star integration. Do not promote or release
`MolToSmilesEnumS` as a public API.

The next work should be:

1. identify the `not_generated_by_shared_pipeline` cases and their missing
   spine stages;
2. choose the first small domain eligible for true unified-reference authority;
3. update the domain manifest only when that authority transition is justified;
4. then run the export gate again.
