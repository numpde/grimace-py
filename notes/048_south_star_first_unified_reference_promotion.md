# South Star First Unified-Reference Promotion Candidate

Branch: `south-star`

Date: 2026-05-21

Task: `South Star 129b: Define first unified-reference promotion`

## Recommendation

The first honest unified-reference promotion should be the single-atom
atom-text domain.

Do not promote directional-stereo, disconnected composition, ring traversal, or
tetrahedral support first. Those domains have richer traversal/constraint
interactions and still rely on temporary witness authority. They may already be
generated by the shared pipeline, but they are not the best first proof of the
authority transition.

## Candidate Domain

Name:

```text
unified_reference_single_atom_atom_text
```

Initial case set:

- `radical_atom_text_hydrogen`
- `radical_atom_text_methyl`
- `radical_atom_text_oxygen`
- `charged_atom_text_chloride`
- `charged_atom_text_ammonium`

Optional later inclusion:

- neutral single-atom organic-subset cases;
- single-atom isotope/map cases once those are fixture-backed;
- `explicit_bracket_hydrogen_h2` only if the domain is widened from
  single-atom to two-atom markerless atom text. It should not be included in
  the initial single-atom promotion.

## Exact Boundary

The first domain should include only molecules satisfying all of these
conditions:

- one connected component;
- exactly one atom;
- zero bonds;
- no query atom;
- no aromatic atom;
- no atom stereo;
- no ring membership;
- atom text supported by `grimace._south_star.atom_text`;
- no unsupported support-gate categories.

The output support for such a molecule is exactly one string: the renderer text
for the single source atom under the declared atom-text policy.

## Completeness Argument

For a single-atom molecule:

- there is exactly one root choice;
- there are no child-order choices;
- there are no branch choices;
- there are no bond tokens;
- there are no ring labels;
- there are no disconnected-fragment orders;
- there are no marker slots;
- there are no stereo-component assignments;
- there are no tetrahedral ligand-order choices.

The declared South Star language for this domain therefore reduces to one
renderer obligation: emit the supported atom text for the only atom. If the
atom-text policy is deterministic and the support gate accepts the molecule,
the support set has cardinality one.

This is a true one-truth reference case because no fixture-local support
enumerator is needed. The graph-native traversal/renderer path and the
independent completeness argument both reduce to the same single event.

## Required Records

The promotion should require these model records to be present and inspectable:

- `SouthStarMoleculeFacts` with `atom_count == 1`, `bond_count == 0`, and
  `fragment_count == 1`;
- `SouthStarAtomTextFields` for the only atom;
- atom-text modifier obligations when charge or radical state requires them;
- one traversal/root record for the only atom;
- one renderer input or renderer event carrying the atom text;
- generation diagnostics with:
  - `fragment_count == 1`;
  - `traversal_skeleton_count == 1`;
  - `marker_slot_count == 0`;
  - `local_assignment_count == 1`;
  - `solved_assignment_count == 1`;
  - `raw_output_count == 1`;
  - `output_count == 1`.

## Migration Criteria

Before changing fixture authority:

1. Add a domain predicate or diagnostic helper that identifies single-atom
   atom-text cases from `SouthStarMoleculeFacts`, not from fixture ids.
2. Add a test that the predicate accepts exactly the intended initial cases and
   rejects two-atom atom-text cases such as `[H][H]` and `[NH3+]C`.
3. Add a completeness test that derives the singleton support from molecule
   facts plus atom-text renderer facts, not from the pinned expected output.
4. Add a manifest authority such as
   `unified_reference_single_atom_atom_text`.
5. Move only the qualifying single-atom cases to that authority.
6. Verify the readiness matrix reports those cases as promoted while all
   temporary-witness and graph-native-regression cases remain blocked.

## Minimal Manifest Change

Add one unified-reference authority:

```text
SOUTH_STAR_SINGLE_ATOM_ATOM_TEXT_UNIFIED_REFERENCE_AUTHORITY =
    "unified_reference_single_atom_atom_text"
```

Then include it in `SOUTH_STAR_UNIFIED_REFERENCE_AUTHORITIES`. Only fixtures
whose domain predicate passes should use that authority.

## Non-Goals

- Do not promote the whole `charged_atom_text` or `radical_atom_text` feature
  areas.
- Do not treat graph-native regression output as final authority.
- Do not promote multi-atom markerless atom-text cases in the first slice.
- Do not use RDKit writer support as the support oracle.

## Follow-Up

Create an implementation task to promote this exact domain. If the
implementation reveals that atom-text rendering lacks a sufficiently explicit
single-event renderer record, split that out as a smaller prerequisite instead
of weakening the authority criteria.
