## Verdict

Do not export `MolToSmilesEnumS` yet.

The current fixture-backed private surface is internally coherent: every
checked case is unified-reference-backed, generated by the shared South Star
spine, and free of case-level public API blockers. That is a real improvement
over the earlier release-readiness audit. It is not the same as package
readiness.

The remaining blockers are now product and surface-boundary blockers rather
than fixture-authority blockers:

- the supported feature surface is still narrow relative to arbitrary SMILES;
- unsupported categories still define a large frontier;
- the public API label, maturity level, and release contract are not decided;
- the semantic enumerator has development benchmark evidence, not a
  user-facing performance story.

## Interpretation

The old failure mode was authority: fixtures passed, but support evidence was
temporary or regression-only. That blocker is cleared for the current checked
surface.

The new failure mode is scope: the checked surface is coherent but still
explicitly incomplete. Exporting now would make users reason about a semantic
enumerator whose fail-fast frontier is still central to normal chemistry
inputs. That can be acceptable only as a deliberately experimental API with a
clear maturity label, not as a stable public sibling of `MolToSmilesEnum`.

## Recommendation

Keep `MolToSmilesEnumS` private for now.

The next export-gate review should happen after at least one of these changes:

1. broaden aromatic coverage beyond markerless monocycles, supported branches,
   and narrow unmodified fused ring systems;
2. reduce the unsupported atom/bond-text frontier;
3. decide the public maturity label and naming contract;
4. write release-facing docs that describe the unsupported frontier as a
   product contract, not just internal diagnostics.

If the project wants an earlier experimental export, create a Decision row
specifically for that choice. The decision should compare:

- private-only continuation;
- explicitly experimental export;
- narrow feature-family export;
- waiting for broader feature-family proof.
