This release hardens the project around containerized development, release
validation, documentation, and RDKit correctness evidence.

Highlights

- Added strict Docker-backed `make` lanes for checks, Rust tests, installed
  package tests, pinned RDKit parity, exact public invariants, packaging, docs,
  and opt-in timings.
- Hardened CI and release workflows with pinned actions, pinned runner images,
  explicit shells, checked-in release notes, trusted PyPI publishing, and
  artifact boundary validation.
- Added release archive validation for wheel and source distribution contents,
  including unsafe paths, special archive members, unexpected top-level wheel
  members, generated outputs, and secret-shaped files.
- Reworked the documentation into a GitHub Pages site with concise top-level
  pages for intro, concepts, runtime behavior, limitations, timings, API, and
  contributor guides.
- Added timing plots, timing environment details, and current benchmark tables
  to the documentation.
- Expanded RDKit correctness evidence with dataset-backed exact support cases,
  deterministic writer-membership cases, fixture provenance counts, serializer
  coverage reporting, and skeptic-grade correctness goals.
- Retired the old random-reference artifact snapshots and removed their loaders,
  policy tests, generated data, and stale helper surfaces.

Validation

- `make ci`
- `make docs`
- `python -m unittest tests.checks.test_docs_pages tests.checks.test_correctness_coverage_report -q`
