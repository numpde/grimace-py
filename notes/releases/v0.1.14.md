This release adds optional zstd compression for serialized `PreparedMol`
payloads, ships trained dictionaries for that storage path, and tightens the
containerized release-support lanes.

Highlights

- Added `PreparedMol.to_bytes(compression="zstd")` support with built-in,
  versioned zstd dictionaries and automatic dictionary selection in
  `PreparedMol.from_bytes(...)`.
- Shipped trained PreparedMol zstd dictionaries with manifests, generator
  tooling, preflight/postflight validation, and archive checks that ensure the
  dictionary data is present in wheels and sdists.
- Added PreparedMol zstd compression-rate tests and timing artifacts comparing
  raw bytes, no-dictionary zstd, and dictionary zstd across compression levels.
- Added Docker-backed lanes for dictionary generation and PreparedMol zstd
  timings, with timing outputs grouped by dictionary artifact.
- Renamed the local package-artifact validation lane to `make test-package`;
  it now builds and validates wheel/sdist artifacts in a container-local
  temporary directory without leaving host `dist/` output.
- Reduced non-stereo determinized decoder memory churn with compact visited
  state storage.
- Reworked docs pages, local docs serving, link checks, timing pages, fixture
  guides, and RDKit serializer coverage explanations.

Validation

- `make checks`
- `make test-package`
