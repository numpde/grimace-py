This release adds public seeded SMILES sampling and exposes branch-count
context on decoder choices.

Highlights

- Added `MolToSmilesSample(...)`, returning one seeded Grimace-supported SMILES
  path with the visible next-token choices recorded at each step.
- Added public `SmilesSample` and `SmilesSampleStep` records for sampled token
  paths, selected tokens, decoder view, sampling mode, and per-prefix branch
  counts.
- Exposed `branch_count` on decoder choices. Branch-preserving decoders report
  `1`; determinized decoders report how many branch-preserving choices are
  represented by the visible token.
- Added deterministic Rust-backed sampling infrastructure with pinned RNG
  behavior and bounded-index sampling.
- Tightened public option handling, sampling mode validation, runtime-walk
  contracts, and public API/docs synchronization tests.
- Refreshed API, runtime, guide, correctness, fixture, and timing
  documentation for the new sampling surface and release behavior.
- Refreshed checked-in timing artifacts.

Validation

- `make ci`
- `make test-package`
- `make docs`
- `python -m unittest tests.contract.test_public_api_docs`
