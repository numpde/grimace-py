This release hardens release validation and fixes two runtime surface-boundary
issues found during audit.

Highlights

- Fixed non-stereo preparation for RDKit molecules that retain residual
  `BondDir` metadata after stereo removal. Grimace now normalizes to the
  selected writer surface, so hidden directional metadata no longer causes
  non-stereo runtime failures.
  For example, this molecule previously raised
  `ValueError: Directional bond tokens are not supported yet` with
  `isomericSmiles=False, allBondsExplicit=False`:

  ```text
  O=S(=O)(/C(c1ccc(Cl)cc1)=c1\o/c(=C\CCBr)c2ccccc12)c1ccc(Br)cc1
  ```

  It now follows the selected non-stereo writer surface and accepts RDKit's
  corresponding rooted non-isomeric serialization:

  ```text
  O=S(=O)(C(c1ccc(Cl)cc1)=c1oc(=CCCBr)c2ccccc12)c1ccc(Br)cc1
  ```
- Tightened decoder terminal-state invariants so accepted prefixes do not keep
  dead outgoing transitions.
- Strengthened release artifact validation for wheel and source distribution
  metadata, native extension placement, archive shape, package-data equivalence,
  duplicate archive members, dictionary artifacts, and source provenance.
- Centralized JSON fixture readers and added earlier validation for serializer,
  support-count, timing, zstd, molecule, and dictionary inputs.
- Clarified runtime documentation around writer surfaces: the complete writer
  flag set determines whether directional/stereo metadata is writer-visible.

Validation

- `make checks`
- `make test`
- `make test-package`
- `make docs`
