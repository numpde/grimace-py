This release hardens the public runtime boundary, PreparedMol storage, and
release posture.

Highlights

- Published release support now covers CPython `3.11` through `3.14` on Linux
  `x86_64`, plus source distributions.
- `PreparedMol` byte loading and zstd compression now fail earlier and more
  defensively:
  - raw PreparedMol payloads are capped at 1 MiB;
  - zstd frames must carry a content size;
  - shipped dictionaries are checked against manifest size, SHA-256, dictionary
    ID, and training level before use.
- Negative `rootedAtAtom` values are normalized to RDKit-style unrooted mode,
  with `-1` remaining the preferred public spelling.
- Public API smoke tests now assert the top-level export surface and installed
  package behavior, including `MolToSmilesSample`.
- Release and container checks now share their policy owners for Python tags,
  dependency pins, workflow action pinning, release notes, and compose
  hardening.
- RDKit fixture evidence and support-count scripts now fail closed on missing
  or out-of-policy fixture roots.

Validation

- `make checks`
- `make test`
- `make test-package`
